Why we built it · design story

VibeStat-Text was not built from a feature checklist alone. An Ed.D. scholar wrote a dissertation centered on text and document analysis through a far more manual path than this tool provides today — learning R, packages, environment setup, and scripts alone, often wanting to quit along the way.

Later, while supervising graduate students in text mining, the same barriers appeared again: time lost to installation and coding instead of corpus design, research questions, and interpretation. That field experience became the direct motive for development.

The goal is research question → corpus → analysis → interpretation with built-in R, buttons, dictionary & preprocess, and a built-in textbook. Optional Vibe-Talk and Vibe-Loom extend interpretation and drafting (your API key).

The app enables richer exploration and reproducibility than the designer had for their own dissertation — but analysis quality and interpretation remain the researcher’s responsibility. It is for research, institutions, and practice — not a student homework shortcut.

R engine · why you can trust it

VibeStat-Text is not an Excel macro or a black-box stats engine. Analysis runs on your PC via R (Rscript.exe). Each button triggers validated R scripts you can inspect in R Code / Console.

Text mining uses tidytext · quanteda · topicmodels · wordcloud2 and other standard academic R packages. With the same packages, preprocessing, and settings, results follow the same R computation path as running scripts in RStudio (numbers may differ if package versions, stopwords, or seeds differ).

  • Local execution — core mining does not send full corpora to cloud LLMs
  • Saved outputs — CSV and images under .vibestat/output/ for papers and reports
  • Reproducibility aids — dictionaries, preprocessing, and session saves for re-runs
  • Korean morphology — elbird (Kiwi), etc.; LIVE LOG / console records the analyzer used
  • Independent product — not an official R Foundation app; R engine use is disclosed with attribution

Unlike Vibe Coding (AI generates scripts for you), VibeStat-Text helps reproducible R-based text mining through a UI. Vibe-Talk and Vibe-Loom are optional interpretation/drafting aids; the core is local R.

What is VibeStatics?

VibeStatics is a Korean software brand for R-based text mining and statistics on Windows. VibeStat-Text is the flagship product: button-driven analysis with a built-in R engine. Also known in Korean as 바이브스타틱스 and 바이브통계 (Vibe Statistics).

Who designed VibeStat-Text and why?

An Ed.D. scholar designed it from a dissertation and graduate text-mining supervision experience. High R entry barriers — experienced firsthand and seen repeatedly in supervision — led to built-in R, buttons, and textbooks so researchers can focus on corpus, questions, and interpretation.

What are Vibe-Talk and Vibe-Loom?

Vibe-Talk — conversational analysis assistant. Vibe-Loom — paper/report drafting support. Optional AI (your key); core mining runs offline with built-in R.

Is VibeStatics the same as Vibe Coding or Naver VIBE?

No. Vibe Coding means generating R/Python scripts with AI. Naver VIBE is a music service. VibeStatics is independent desktop research software for corpus-based text mining.

Is it “Five Statics” (the number 5)?

No. The name is Vibe + Statistics → Vibe Statics, matching the domain vibestatics.com.

What is VibeStat-Text?

Windows text mining for research, institutions, practice, and teaching. PDF · Excel · Word · HWP · CSV · TXT → frequency · word clouds · LDA · TF-IDF · document comparison. Built-in R runs locally; an independent product, not an official R app. R · trust →

Is it the same as running R directly?

Core analysis runs via R (Rscript) with tidytext · quanteda · topicmodels and similar packages. Buttons are the UI; you can view R code and console output. Same packages, preprocessing, and settings → same R computation path as RStudio scripts.

Is the English UI available?

Korean UI is complete and ready for research and teaching. The English program UI is in active development — this English web page helps global researchers find VibeStatics while the app UI is being localized.

Do I need to code in R?

Core workflows use buttons — no tm/ggplot scripts required for frequency, word clouds, LDA, or document comparison. The R engine runs locally inside the app. Advanced users can still view R code and console.

Do I need an API key?

No for core text mining — runs offline with built-in R. Optional AI (Vibe-Talk, Vibe-Loom) requires your own API key (BYOK).

Can I use VibeStat-Text without RStudio?

Yes. Run R text mining and statistical analysis on Windows via buttons without installing RStudio separately. The app includes an R engine and packages such as tidytext · quanteda · ggplot2 · topicmodels on your PC. R · trust →

Which R packages are used?

Standard academic packages: tm · tidytext · quanteda · topicmodels · wordcloud2 · ggplot2. Computation runs via Rscript; LIVE LOG and console show what was used.

Is there a free word cloud demo?

Yes. The Word Cloud Playground on the Korean homepage lets you paste text and save a PNG without installing software. Morphological analysis, LDA, and document comparison need the VibeStat-Text app.

Is it for statistical and text analysis?

Yes. For papers, policy reports, surveys, interviews, and document corpora — frequency, word clouds, LDA topics, TF-IDF, document comparison. For researchers, graduate students, faculty, institutions, and professionals.
